Isolation

Each feature runs in its own git worktree. Changes are isolated. Phases run in parallel as OS processes.

The Consensus Loop

Raft-inspired verification: no agent is judge, jury, and executioner.

IMPLEMENT
Sonnet (Leader)
Picks next pending story from prd.json → implements with TDD → commits → outputs STORY_IMPLEMENTED:US-003
VERIFY ×3 (parallel)
Haiku (Followers)
Each independently: check acceptance criteria against actual code (not commit msg) → run quality checks → cast VOTE:PASS or VOTE:FAIL:{reason}
TALLY
Bash (deterministic)
2/3 PASS → committed  |  2/3 FAIL → rejected + feedback  |  Tie → Opus tiebreaker

State Machine

pending → implementing → verifying → passed or rejected → retry up to 3x → blocked

Raft Safety Properties

Distributed consensus principles applied to AI agent orchestration.

Election Restriction → Quality Gate

Agent must have latest git pull before implementing. No stale code.

Current-Term Commits → No Stale Approvals

Votes only count from the current term. Code change invalidates all prior votes.

Majority Quorum → Byzantine Tolerance

With 3 verifiers, 1 can hallucinate "PASS" and you still catch it. Cheaper than making one agent smarter.

nextIndex Backtracking → Structured Retry

FAIL reasons feed into next attempt. No blind retry — implementer reads exactly why it was rejected.

Split Brain Prevention → No Conflicting States

One leader implements at a time. Verifiers only verify current term. No race conditions on prd.json. We take the consensus & commit protocol from Raft, not the distributed infrastructure.

Why Superpowers Can't Do This

No Automation

Human must manually choose the model for each subagent dispatch. Sugar Rush automates model selection per phase.

No Escalation Logic

If a cheap model fails, the human must notice and re-dispatch. Our loop detects failures and auto-escalates to Opus.

No Structured Stories

Plans are prose — cheap models have more room to misinterpret. Our prd.json is machine-readable with clear acceptance criteria.

No Loop

Each task is one-shot dispatch. No mechanism to retry with a bigger model. Sugar Rush retries with structured feedback automatically.

Superpowers cannot “leave it running overnight.”
Sugar Rush can — and will come back with 100 verified, committed stories.
