# Feature Brainstorming

## Iron Law
`DIVERGE BEFORE YOU CONVERGE — NO FILTERING DURING IDEATION`

## Phases

### 1. Diverge
Generate 10+ ideas without judgment. Quantity over quality. Wild ideas welcome.
- Write every idea down, no matter how impractical
- Build on previous ideas — "yes, and..." not "no, but..."
- Set a minimum: 10 ideas before moving to the next phase
- Include at least 2 "impossible" ideas — they often spark practical ones

### 2. Cluster
Group related ideas into themes. Name each cluster.
- Look for patterns and connections between ideas
- An idea can belong to multiple clusters
- Name each cluster with a descriptive label
- Identify which clusters overlap or conflict

### 3. Evaluate
Score each cluster on three dimensions:
- **Feasibility** (1-5): Can we actually build this?
- **Impact** (1-5): How much value does this deliver?
- **Effort** (1-5): How much work is required?

Rank clusters by **impact/effort ratio**. High impact + low effort = quick wins. High impact + high effort = strategic bets.

### 4. Converge
Select the top 3 ideas/clusters. For each:
- Write a 2-sentence description of what it does and why it matters
- List 3 key risks (technical, user adoption, scope creep)
- Define one concrete next step (specific enough to start immediately)

## Red Flags — If You Catch Yourself Thinking:

| Thought | Reality |
|---|---|
| "This idea won't work because..." | You're filtering during diverge. Write it down anyway. |
| "We only need 3-4 ideas" | 10+ minimum. The best ideas come after the obvious ones. |
| "Let me evaluate as I go" | Separate diverge from converge. Evaluate AFTER generating. |
| "This is too wild/impractical" | Wild ideas spark practical ones. Capture everything. |

## Rules
- Generate at least 10 ideas before evaluating any
- No criticism during the diverge phase
- Each idea gets equal initial weight during diverge
- Output a ranked list with concrete next steps for the top 3
- Document all ideas, even rejected ones — they're future inspiration
